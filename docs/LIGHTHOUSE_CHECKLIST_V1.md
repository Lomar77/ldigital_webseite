# L-Digital Lighthouse-Checkliste V1

Diese Checkliste bereitet den spaeteren Lighthouse-/Performance-Check der statischen L-Digital Website vor. Sie ist eine Pruefgrundlage. Es werden dadurch keine Optimierungen, Code-Aenderungen, Bildkomprimierungen oder Tracking-Einbindungen vorgenommen.

## Performance

- [ ] Bildgroessen pruefen.
- [ ] Dateigroessen der HTML-Dateien pruefen.
- [ ] CSS-Groesse pruefen.
- [ ] Ungenutzte Dateien pruefen.
- [ ] Ladezeit auf echter Zielumgebung pruefen.
- [ ] Mobile Performance pruefen.

## Accessibility

- [ ] Alt-Texte bei Bildern pruefen.
- [ ] Kontraste pruefen.
- [ ] Ueberschriftenstruktur pruefen.
- [ ] Buttons und Links sind gut erkennbar.
- [ ] Tastaturbedienbarkeit grob pruefen.

## SEO

- [ ] Seitentitel pruefen.
- [ ] Meta-Beschreibungen pruefen.
- [ ] H1-Struktur pruefen.
- [ ] Sprechende URLs pruefen.
- [ ] `sitemap.xml` pruefen.
- [ ] `robots.txt` pruefen.

## Best Practices

- [ ] HTTPS auf Zielumgebung pruefen.
- [ ] Keine Konsolenfehler.
- [ ] Keine kaputten Links.
- [ ] Keine alten WordPress-Reste.
- [ ] Keine Tracking-Skripte ohne Entscheidung.

## Mobile

- [ ] Darstellung auf Smartphone pruefen.
- [ ] Keine horizontalen Scrollbalken.
- [ ] Buttons gut klickbar.
- [ ] Texte gut lesbar.
- [ ] Layout bleibt stabil.

## Noch nicht ausfuehren

Der Lighthouse-Test soll erst auf einer echten Staging- oder Zielumgebung ausgefuehrt werden, nicht nur lokal. Optimierungen werden erst nach gemessenen Ergebnissen gemacht.
