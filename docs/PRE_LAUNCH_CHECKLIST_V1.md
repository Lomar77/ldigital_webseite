# Pre-Launch-Checkliste v1.0

## Zweck

Diese Checkliste beschreibt, was vor einer Veroeffentlichung der L-Digital Website noch geprueft oder erledigt werden muss.

## Recht / Pflichtseiten

- [ ] Impressum vorhanden.
- [ ] Zustelladresse eingetragen.
- [ ] Datenschutzseite vorhanden.
- [ ] Finale rechtliche/praktische Gesamtpruefung offen.
- [ ] Keine rechtliche Freigabe behaupten.

## Marke / Assets

- [ ] Logo finalisieren.
- [ ] Favicon finalisieren.
- [ ] Social Preview / Open Graph Bild finalisieren.
- [ ] Keine weiteren KI- oder Codex-Bildaenderungen, solange Logo manuell blockiert ist.

## Technik / Website

- [ ] Alle HTML-Seiten pruefen.
- [ ] CSS pruefen.
- [ ] Keine Layout-Brueche.
- [ ] Desktop-Darstellung pruefen.
- [ ] Mobile-Darstellung pruefen.
- [ ] Interne Links pruefen.
- [ ] Impressum/Datenschutz-Links pruefen.
- [ ] Kontaktlinks pruefen.
- [ ] 404-Seite pruefen.

## SEO / Auffindbarkeit

- [ ] Seitentitel pruefen.
- [ ] Meta-Beschreibungen pruefen.
- [ ] H1-Struktur pruefen.
- [ ] `sitemap.xml` pruefen.
- [ ] `robots.txt` pruefen.
- [ ] Google Search Console spaeter einrichten.
- [ ] Google Analytics aktuell nicht einbauen.

## Performance / Lighthouse

- [ ] Lighthouse-Test auf Staging- oder Zielumgebung durchfuehren.
- [ ] Performance pruefen.
- [ ] Accessibility pruefen.
- [ ] Best Practices pruefen.
- [ ] SEO pruefen.
- [ ] Bildgroessen pruefen.
- [ ] Alt-Texte pruefen.

## Hosting / Deployment

- [ ] WordPress-Backup erstellen.
- [ ] WordPress-Datenbank sichern.
- [ ] Upload-Ordner sichern.
- [ ] `.htaccess` sichern.
- [ ] `robots.txt` und `sitemap.xml` sichern.
- [ ] Zielordner auf Hosting pruefen.
- [ ] Domain-Zuordnung pruefen.
- [ ] HTTPS pruefen.
- [ ] Statische Website zuerst auf Staging/Zielumgebung testen.
- [ ] Rollback-Plan bereithalten.

## Launch-Entscheidung

Vor Livegang muessen mindestens diese Punkte erfuellt sein:

1. Logo/Favicon/Social Preview entweder final oder bewusst als nicht final akzeptiert.
2. Impressum und Datenschutz final geprueft.
3. Mobile/Desktop geprueft.
4. Links geprueft.
5. Lighthouse/Performance geprueft.
6. Backup vorhanden.
7. Rollback moeglich.
8. Website auf Zielumgebung getestet.

## Nicht vor Launch vergessen

- [ ] Keine echten Finanzdaten oder L-BWoods-Dateien im public Website-Repo.
- [ ] Keine lokalen Windows-Pfade.
- [ ] Keine alten WordPress-Links.
- [ ] Keine unnoetigen Tracking-Tools.
- [ ] Keine falschen rechtlichen Aussagen.

## Status

Die Website befindet sich in der Pre-Launch-Phase. Ein Livegang ist erst sinnvoll, wenn Logo/Assets, rechtliche Endpruefung, Staging-Test, Lighthouse-Test und Deployment-Vorbereitung abgeschlossen sind.
