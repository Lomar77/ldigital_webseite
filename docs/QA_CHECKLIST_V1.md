# L-Digital QA-Checkliste V1

Diese Checkliste dient der manuellen Desktop-/Mobile-Pruefung der statischen L-Digital Website. Sie ist eine technische und inhaltliche Pruefgrundlage, keine rechtliche Freigabe und keine Lighthouse-Optimierung.

## Desktop-Pruefung

- [ ] Header ist sichtbar und sauber ausgerichtet.
- [ ] Hauptnavigation ist sichtbar, nachvollziehbar und funktioniert.
- [ ] Hero-Bereich wirkt vollstaendig und nicht abgeschnitten.
- [ ] Leistungs-Cards sind sauber lesbar.
- [ ] CarDesk-Bereich wirkt als Konzept/Pilot und nicht wie ein fertiges Produkt.
- [ ] Kontaktbereich ist klar und die naechste Handlung ist verstaendlich.
- [ ] Footer enthaelt funktionierende Links zu Impressum und Datenschutz.
- [ ] Keine offensichtlichen Layout-Brueche, Ueberlagerungen oder ungewollten Abstaende sichtbar.

## Mobile-Pruefung

- [ ] Website ist auf Smartphone-Breite gut lesbar.
- [ ] Navigation ist auf kleiner Breite nutzbar.
- [ ] Buttons und Links sind gut klickbar.
- [ ] Cards sind sauber untereinander gestapelt.
- [ ] Texte sind nicht zu breit, zu klein oder abgeschnitten.
- [ ] Keine horizontalen Scrollbalken sichtbar.
- [ ] Abstaende wirken sauber und konsistent.
- [ ] Footer funktioniert auf kleiner Breite und bleibt lesbar.

## Link-Pruefung

- [ ] Interne Navigation funktioniert.
- [ ] Links zu Impressum und Datenschutz funktionieren.
- [ ] Kontaktlinks funktionieren, insbesondere `mailto:info@ldigital.ch`.
- [ ] Keine toten lokalen Datei- oder Bildpfade vorhanden.
- [ ] Keine Links fuehren auf eine alte WordPress-Struktur.

## Recht-/Launch-Pruefung

- [ ] Impressum ist vorhanden.
- [ ] Datenschutzseite ist vorhanden.
- [ ] Zustelladresse ist eingetragen.
- [ ] Keine Google-Analytics-Skripte vorhanden.
- [ ] Google Search Console kann spaeter ergaenzt werden.
- [ ] Finale rechtliche und praktische Gesamtpruefung vor Launch bleibt offen.

## Performance-/Lighthouse-Vorbereitung

Nur pruefen und dokumentieren, keine Optimierung direkt umsetzen.

- [ ] Bildgroessen pruefen.
- [ ] Alt-Texte pruefen.
- [ ] Seitentitel und Meta-Beschreibungen pruefen.
- [ ] Mobile Darstellung pruefen.
- [ ] Accessibility-Basics pruefen.
- [ ] Performance spaeter auf Staging oder Zielumgebung messen.
