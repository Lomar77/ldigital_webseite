# Logik V1

Jede finanzielle Bewegung wird mit drei Fragen erfasst:

1. Was ist passiert?
2. Womit wurde bezahlt?
3. Gibt es einen Beleg?

Diese Fragen reichen fuer Version 1.0 aus, um Kapital, Einnahmen, Ausgaben und Belege sauber auseinanderzuhalten.

## Bewegungsarten

### Einnahme

Eine Einnahme ist Geld, das L-Digital durch eine Leistung, ein Produkt oder eine andere geschaeftliche Quelle erhaelt.

Beispiel: Ein Kunde bezahlt eine Rechnung.

### Ausgabe

Eine Ausgabe ist Geld, das fuer L-Digital ausgegeben wird.

Beispiel: Bueromaterial, Software, Werkzeug oder externe Dienstleistungen.

### Bareinlage

Eine Bareinlage ist Kapital, das vom Inhaber oder aus privater Quelle in L-Digital eingebracht wird.

Wichtig: Eine Bareinlage ist kein Umsatz und kein Gewinn. Sie erhoeht nur das verfuegbare Kapital.

### Privat bezahlte Geschaeftsausgabe

Eine privat bezahlte Geschaeftsausgabe ist geschaeftlich relevant, wurde aber mit einem privaten Zahlungsmittel bezahlt.

Beispiel: Bueromaterial wird mit einer Privatkarte gekauft, gehoert aber zu L-Digital.

Diese Ausgabe muss dokumentiert werden, damit spaeter nachvollziehbar bleibt, was privat fuer das Unternehmen bezahlt wurde.

### Rueckzahlung an Inhaber

Eine Rueckzahlung an den Inhaber ist eine Ausgleichszahlung von L-Digital an die private Person, zum Beispiel fuer vorher privat bezahlte Geschaeftsausgaben.

Sie ist keine neue Ausgabe, wenn die urspruengliche Ausgabe bereits erfasst wurde.

### Umbuchung

Eine Umbuchung verschiebt Geld zwischen Zahlungsmitteln oder Konten, ohne dass neuer Umsatz oder eine neue Ausgabe entsteht.

Beispiel: Geld wird von Kasse auf Bankkonto verschoben.

## Grundregel

Nicht jede Geldbewegung ist Umsatz. Nicht jede Zahlung ist Gewinn oder Verlust. L-BWoods trennt deshalb Kapital, Einnahmen, Ausgaben und Umbuchungen bewusst voneinander.
